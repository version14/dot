# v0.5 — Project as Code (PaC)

> Status: not started. Requires v0.3 (conflict resolution) to be stable first.

---

## Goal

Let teams describe their entire project — or monorepo — in a single `dot.yaml` file.
Declarative, diffable, reviewable in a PR.

---

## What ships

### dot.yaml input layer

An alternative to the TUI. Same engine, different input.

```yaml
meta:
  name: my-platform
  type: monorepo

defaults:
  config:
    ci: github-actions
    linter: golangci-lint

apps:
  api:
    language: go
    type: rest-api
    modules: [rest-api, postgres, auth-jwt]
    config:
      deployment: docker
      monitoring: grafana

  frontend:
    language: typescript
    type: nextjs
    modules: [shadcn-ui, tanstack-query]
    config:
      deployment: vercel
```

### dot plan

```bash
dot plan    # diff between dot.yaml and .dot/config.json
```

Shows what will change: added apps, removed apps, added/removed modules per app.

### dot apply

```bash
dot apply   # run the delta from dot plan
```

Runs generators for the delta, pipes through the conflict-aware pipeline.
