# v1.1 — MCP server

> Status: direction only. Design starts after v1.0 ships.

---

## Goal

Let AI agents scaffold and extend projects via the MCP protocol.
Same engine, same generators, same Spec — new input layer.

---

## What ships

### dot-mcp

A standalone MCP server that exposes dot's capabilities to any MCP-compatible agent.

```
Agent → MCP call → dot-mcp → Spec → Generator engine → FileOp pipeline → project on disk
```

The MCP server is a thin input layer. It parses the agent's intent into a Spec and hands it
to the same engine the CLI uses. No new generators needed.

### MCP tools (proposed)

| Tool | What it does |
|------|-------------|
| `dot_init` | Scaffold a new project from a Spec description |
| `dot_add_module` | Add a module to an existing dot project |
| `dot_list_generators` | List available generators and modules |
| `dot_status` | List files with unresolved conflicts |

---

## Why v1.1 and not earlier

The MCP server is an input layer — it needs the engine to be stable first.
Building it before v1.0 means building on moving ground.
