# v0.1 — What ships today

---

## Status: shipped

---

## What works

| Command | What it does |
|---------|-------------|
| `dot init` | huh TUI survey → Spec → GoRestAPIGenerator → project on disk + .dot/config.json |
| `dot new route <name>` | Adds a route to a Go REST API project |
| `dot new handler <name>` | Adds a handler stub |
| `dot help` | Lists available commands from .dot/config.json |
| `dot version` | Prints the binary version |
| `dot self-update` | Updates the binary to the latest GitHub release |

## What is NOT in v0.1

- `dot add module` — conflict strategy not yet decided
- Any generator other than GoRestAPIGenerator
- dot.yaml input layer
- Community generator loading

## Known constraints

- AnchorImportBlock does not support blank imports, build-tagged imports,
  or multiple single-line imports. See patch-strategies.md.
- Only Go projects are supported.
- .dot/manifest.json is written but conflict detection is not implemented yet.
