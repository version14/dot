# v1.0 — Full custom project creation

> Status: not started. Requires v0.2 through v0.5 to be complete and stable.

---

## Goal

Everything works together. A developer can take a project from zero to production-ready
scaffold using any combination of official generators, community generators, and PaC —
and extend it safely over time.

---

## What v1.0 means

v1.0 is not a new feature release. It is the milestone where the full loop works end to end:

1. `dot init` or `dot.yaml` → scaffold any project type
2. `dot add module` → extend safely, conflicts surfaced and documented
3. `dot generator install` → community generators work reliably
4. `dot plan` + `dot apply` → declarative project management for teams
5. Distribution is stable: Homebrew, curl, go install, self-update all work

---

## Stabilisation work

- Polish the TUI survey flow (all question types, validation, error states)
- Harden conflict resolution edge cases (binary files, deleted files, renamed files)
- Generator versioning: pin generator versions in `.dot/generators.json`, lockfile
- `dot reinit` — re-run dot init on an existing project with full conflict detection
- Comprehensive documentation for generator authors
- Public registry stable and searchable

---

## Definition of done

A developer who has never used dot can:
1. Install it in one command
2. Scaffold a production-ready Go API + Docker + GitHub Actions in under 5 minutes
3. Add Postgres a week later without breaking anything
4. Share the dot.yaml with their team and reproduce the same project from scratch
