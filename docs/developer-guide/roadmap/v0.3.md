# v0.3 — dot add module + conflict resolution

> Status: not started. Blocked on open-decisions.md #2.

---

## Goal

Deliver the core promise: **safe extension of an existing project**.
A project created with dot can absorb new modules without breaking what's already there.

---

## What ships

### dot add module

```bash
dot add module postgres     # add Postgres to an existing dot project
dot add module redis        # add Redis
dot add module docker       # add Docker
```

Looks up the generator for the requested module, runs `Apply()`, pipes the ops through
the conflict-aware pipeline.

### Conflict detection

Uses `.dot/manifest.json` hashes to detect user-modified files:
- File hash matches manifest → unmodified → apply op directly
- File hash differs → user modified the file → write conflict markers

### Conflict markers

Git-style markers written directly into the affected file:

```
<<<<<<< dot (postgres generator)
db, err := sql.Open("postgres", os.Getenv("DATABASE_URL"))
=======
// your existing code
>>>>>>> current
```

### dot status

```bash
dot status    # list files with unresolved conflicts
```

### dot resolve

```bash
dot resolve   # mark all conflicts resolved, update manifest hashes
```
