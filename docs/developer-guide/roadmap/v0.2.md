# v0.2 — Generators + local custom generators

> Status: not started.

---

## Goal

Make dot genuinely useful for Go projects beyond the basic REST API stub.
Let users write their own generators without publishing anything.

---

## What ships

### Official generators

| Generator | Module | What it scaffolds |
|-----------|--------|------------------|
| GoPostgresGenerator | `postgres` | Connection setup, migration runner, repository interface |
| GoRedisGenerator | `redis` | Client setup, cache helper |
| GoAuthJWTGenerator | `auth-jwt` | JWT middleware, token helpers |
| GoGRPCGenerator | `grpc` | Proto setup, server stub |
| GitHubActionsGenerator | `github-actions` | CI workflow for Go projects |
| DockerGenerator | `docker` | Dockerfile + .dockerignore |

### Local custom generators

Users can write a generator in Go, point dot at it, and use it immediately — no publishing required.

```bash
dot generator add ./my-generator    # register a local generator
dot generator list                  # show registered generators
dot generator remove my-generator   # unregister
```

Local generators implement the same `Generator` interface as official ones.
They are stored in `.dot/generators.json`.
